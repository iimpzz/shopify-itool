# vehicle11 配置页页面结构稿

## 1. 页面目标

这个页面只做三件事：

1. 维护动态字段图片
2. 维护静态分类与静态卡片
3. 维护分类图标与少量高级字段

页面必须简洁、分层、无横向滚动。

---

## 2. 页面总结构

整页按纵向 6 层组织：

1. 页面头部
2. 主模块切换
3. 当前模块列表区
4. 当前条目详情区
5. 高级设置区
6. 校验与 JSON 区

SaveBar 固定使用 Shopify SaveBar，不单独做底部保存按钮条。

---

## 3. 页面头部

### 3.1 头部内容

- 标题：`vehicle11 页面配置`
- 副标题：`维护字段图片、静态分类和分类图标`
- 辅助说明：`修改后会整份回写 pages-vehicle-json`

### 3.2 头部信息

只保留 3 个轻量信息块：

- 分类数
- 卡片数
- 字段图片数

不要放太多统计卡，不要做大仪表盘。

---

## 4. 主模块切换

主模块只保留两个一级入口：

- `字段图片`
- `静态分类`

`高级设置` 不作为一级 Tab，而是作为当前内容页里的折叠区。

这样用户一眼就能看懂，主流程始终只有两条。

---

## 5. 模块一：字段图片

这个模块对应：

- `location_assets`

### 5.1 结构

分成两层：

1. 字段列表
2. 字段详情

### 5.2 字段列表

每个列表项只显示：

- 字段名 `key`
- desktop 是否已配
- mobile 是否已配
- alias 数量

列表区操作：

- 新增字段
- 删除字段
- 搜索字段

### 5.3 字段详情

详情区字段：

- `key`
- `aliases`
- `desktop`
- `mobile`

详情区布局：

- 基础信息一块：`key`、`aliases`
- 图片信息一块：`desktop`、`mobile`
- 两张图各自带预览

### 5.4 空状态

没有条目时显示：

- 空状态说明
- 一个“新增字段图片”按钮

---

## 6. 模块二：静态分类

这个模块对应：

- `categories`

### 6.1 结构

分成四层：

1. 分类列表
2. 当前分类基础信息
3. 当前分类卡片列表
4. 当前卡片详情

这是整个页面最核心的区域。

### 6.2 分类列表

每个分类项显示：

- `title`
- `slug`
- 卡片数量

列表区操作：

- 新增分类
- 删除分类
- 分类上下排序

### 6.3 当前分类基础信息

只放最核心字段：

- `slug`
- `title`
- `subtitle`

不把 `results_url` 放在这里，避免主区过重。

### 6.4 当前分类卡片列表

这里直接对应前台的：

- `static-vehicle__product-card`

卡片数量来自：

- `categories[*].cards[*]`

所以这里必须支持：

- 新增卡片
- 删除卡片
- 上下排序

每个卡片项显示：

- `title`
- `id`
- 是否有图片
- 是否有链接

### 6.5 当前卡片详情

详情字段：

- `id`
- `title`
- `url`
- `image`
- `mobile_image`

推荐顺序：

1. 先填 `id`
2. 再填 `title`
3. 再填 `url`
4. 最后补 `image` / `mobile_image`

### 6.6 空状态规则

- 空分类允许临时存在
- 但应显示提示：`当前分类还没有卡片`
- 保存时给出非阻断提示或弱阻断提示，建议至少保留一张卡片

---

## 7. 高级设置区

高级设置跟随当前模块显示，但默认折叠。

### 7.1 字段图片模块下

字段图片模块下不需要额外高级设置。

### 7.2 静态分类模块下

高级设置只放两类内容：

- `results_url`
- 分类图标

### 7.3 `results_url`

只跟随当前选中的静态分类显示。

位置：

- 放在分类基础信息下面
- 使用折叠卡片或 `details` 区域

### 7.4 分类图标

产品层面只叫：

- `分类图标`

UI 不区分：

- 静态图标
- 全局图标映射

但实现层要兼容：

- 动态分类写 `category_icons`
- 静态分类同步写 `categories[*].active_icon/default_icon`
- 静态分类同步写 `category_icons[slug]`

### 7.5 动态分类图标列表

建议预置：

- `led-bulbs`
- `lighting-assemblies`
- `accessories-kits`

每项只放两张图：

- `active`
- `default`

---

## 8. 校验与 JSON 区

这个区域放在页面底部，不抢主流程视觉焦点。

### 8.1 校验结果

分成两组：

- 错误
- 提示

错误示例：

- slug 为空
- title 为空
- card id 重复
- URL 非法

提示示例：

- 某分类没有卡片
- 某条目只填了 desktop
- 某分类没填 `results_url`

### 8.2 JSON 面板

默认只读。

支持：

- 复制 JSON
- 查看格式化后的最终结果

不建议默认开放 raw JSON 编辑。

---

## 9. 交互流

### 9.1 字段图片流

1. 点击 `字段图片`
2. 选择一个字段或新增字段
3. 编辑 `key / aliases / desktop / mobile`
4. 页面出现未保存状态
5. 点击 SaveBar 保存

### 9.2 静态分类流

1. 点击 `静态分类`
2. 选择分类或新增分类
3. 编辑分类基础信息
4. 选择卡片或新增卡片
5. 编辑卡片详情
6. 如有需要展开高级设置填写 `results_url` 或图标
7. 点击 SaveBar 保存

---

## 10. 布局要求

### 10.1 桌面端

默认采用纵向分层。

建议：

- 列表区在上
- 详情区在下

超宽屏下才允许局部双列：

- 左侧列表
- 右侧详情

### 10.2 移动端

全部单列。

要求：

- 不出现横向滚动
- 输入框宽度不溢出
- 图片预览自适应
- 按钮满足可点击面积

---

## 11. 视觉要求

整体风格：

- 简约
- 轻后台
- 不是重运营面板

建议视觉语言：

- 浅底卡片
- 弱阴影
- 清晰分组边界
- 当前选中项高亮

避免：

- 太多大色块
- 太多说明文案
- 太多顶部摘要卡

---

## 12. 对应到代码的改造方向

当前主要改造文件：

- `app/components/VehicleImagesEditor.jsx`

建议拆成这几个逻辑区：

1. 页面头部区
2. 模块切换区
3. `location_assets` 列表与详情区
4. `categories` 列表区
5. 当前分类详情区
6. 当前卡片列表与详情区
7. 高级设置区
8. 校验与 JSON 区

如果继续拆组件，优先拆这几个：

- `VehiclePageHeader`
- `VehicleSectionTabs`
- `LocationAssetsPanel`
- `StaticCategoriesPanel`
- `CategoryCardsPanel`
- `VehicleAdvancedPanel`
- `VehicleValidationPanel`

---

## 13. 这份结构稿的落地结论

后续 UI 改造时，必须坚持这几个约束：

- 主流程只有 `字段图片` 和 `静态分类`
- `static-vehicle__product-card` 数量必须可配置增减
- `results_url` 和分类图标进入高级设置
- 页面必须纵向分层，不允许再做成横向拥挤编辑台
- 未开放字段继续保留，不因为 UI 简化而丢数据
